Let \(S=\{a_1<\cdots<a_n\}\subset \mathbb Z\).

A cousin set \(T\) is obtained by moving each \(a_i\) to either \(a_i-1\) or \(a_i+1\), with:

- no image landing back in \(S\),
- no two images equal.

So everything is controlled by the gaps \(a_{i+1}-a_i\).

### 1. Local behavior
- If \(a_{i+1}-a_i=1\), then \(a_i\) must move left and \(a_{i+1}\) must move right.
- If \(a_{i+1}-a_i=2\), then they cannot move toward each other (that would collide at the midpoint).
- If \(a_{i+1}-a_i\ge 3\), the choices are independent across that gap.

So split \(S\) into components separated by gaps \(\ge 3\). The total number of cousins is the product of the numbers for the components.

### 2. Count for one component
Inside one component, all gaps are \(1\) or \(2\).

- If there is a gap \(1\), that forces directions; then every neighboring gap \(2\) propagates the forcing outward.  
  Hence:
  - exactly one gap \(1\) \(\Rightarrow\) exactly **1** cousin set;
  - two gap \(1\)'s \(\Rightarrow\) impossible, so **0** cousin sets.

- If there are no gap \(1\)'s, the component is
  \[
  \{a,a+2,\dots,a+2(m-1)\}
  \]
  of size \(m\). Then valid moves are: first \(k\) elements go left, remaining \(m-k\) go right, for \(k=0,1,\dots,m\).  
  So this component contributes exactly \(m+1\) cousin sets.

Thus every useful component contributes either \(1\) or \(m+1\).

### 3. Optimization
To get exactly \(4040\), only the \(m+1\) factors matter. If a factor \(m+1\) is composite, say \(m+1=ab\) with \(a,b>1\), then replacing that one component by two components of sizes \(a-1\) and \(b-1\) gives the same count \(ab\) but uses fewer elements:
\[
(a-1)+(b-1)=a+b-2<ab-1=m.
\]
So the minimum occurs when all factors are prime.

Prime factorization:
\[
4040=2^3\cdot 5\cdot 101.
\]
Hence the minimum size is
\[
(2-1)+(2-1)+(2-1)+(5-1)+(101-1)=1+1+1+4+100=107.
\]

\[
\{0,2,\dots,198\}\cup\{300,302,304,306\}\cup\{400\}\cup\{500\}\cup\{600\}.

101\cdot 5\cdot 2\cdot 2\cdot 2=4040.
