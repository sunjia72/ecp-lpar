where \(\{a_1,\dots,a_n\}=\{1,\dots,n\}\).

For each \(m\ge 2\), the new value \(a_m\) differs by \(1\) from some earlier value.

Now look at the set of values in each prefix:
\[
S_m=\{a_1,\dots,a_m\}.
\]

- \(S_1=\{a_1\}\), an interval.
- If \(S_{m-1}\) is an interval \([L,R]\), then \(a_m\) must be adjacent to some element already in \(S_{m-1}\).
- Since all values are distinct, the only possible new adjacent values are \(L-1\) or \(R+1\).

So every \(S_m\) must be an interval, and each step extends the current interval either one step left or one step right.

Conversely, any process built this way clearly satisfies the condition.

So to build such a permutation:

1. Choose the first value \(a_1=s\in\{1,\dots,n\}\).
2. Then the interval must expand from \(\{s\}\) to all of \(\{1,\dots,n\}\).
   - Left expansions needed: \(s-1\)
   - Right expansions needed: \(n-s\)

\[
\binom{n-1}{s-1}.
\]

\[
\sum_{s=1}^n \binom{n-1}{s-1}
= \sum_{j=0}^{n-1}\binom{n-1}{j}
