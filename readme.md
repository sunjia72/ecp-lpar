## Formally Solving Answer-Construction Problems in Lean (Repository for Artifact Evaluation)


## Structure
The main repository layout is:

```text
data/
|-- dataset/
|   |-- matharena.json  # MathArena: AIME 2026, HMMT Feb 2026, APEX 2026
|   `-- putnam.json     # PutnamBench
`-- prompts/            # Prompt templates

src/
|-- ecp/                # Main ECP workflow
`-- goedel/             # Proof generation and verification

Formalization/          # Lean project, utilities, REPL, and solved proofs
```

## Requirements & Setup
First, clone the repo.
```bash
git clone https://github.com/sunjia72/ecp-lpar
cd ecp-lpar
```
We suggest using `uv` to build environments (the fast Python package manager from Astral), which can be installed by `curl -LsSf https://astral.sh/uv/install.sh | sh`. The installation can be verified by `curl -LsSf https://astral.sh/uv/install.sh | sh`.

We suggest to use Python 3.12 on Linux / Ubuntu systems. To run answer-construction via commercial models, it suffices to allocate 1 CPU. To run the end-to-end pipeline including theorem-proving, there requires at least 1 GPU (CUDA >= 12.2) with at least 48 GB memory.

The environment can be set up by
```bash
uv venv ecp --python 3.12
source ecp/bin/activate
uv pip install "torch==2.8.0"
uv pip install psutil
uv pip install -r requirements.txt
uv pip install flash-attn --no-build-isolation
```

We use Lean 4.24.0 for proof verification. To set it up, run

```bash
cd Formalization
lake update
lake build Mathlib
lake build utils
lake build REPL
```

To ensure REPL executes correctly, run
```bash
lake exe repl
```

After the set up, run test
```bash
echo '{"path": "test.lean", "allTactics": true}' | lake exe repl
```
Finally, go back to the working directory
```bash
cd ..
```


## Model Download & API Setup
To run the commercial model, it is necessary to set up the OpenAI API key to the environment variable. The key can be generated by clicking "Create new secret key" in https://platform.openai.com/api-keys. Then set the key as follows:
```bash
export OPENAI_API_KEY=<your api key>
```

Alternatively, openrouter API is also allowed, in case OpenAI API is not available:
```bash
export OPENROUTER_API_KEY=<your api key>
```

To run the prover model `Goedel-LM/Goedel-Prover-V2-8B` and `Goedel-LM/Goedel-Prover-V2-32B`, download them through the following commands. This takes around 64GB storage for the 32B model, and 16GB storage for the 8B model. This is mandatory for end-to-end experiment reproduction, but optional for demo (as we can temporarily use commercial model for proving for demo purpose).
```bash
python3 -c "import transformers; transformers.pipeline('text-generation', model='Goedel-LM/Goedel-Prover-V2-32B')"
python3 -c "import transformers; transformers.pipeline('text-generation', model='Goedel-LM/Goedel-Prover-V2-8B')"
```

## Dataset Preparation
We have already built the datasets to `data/datasets`. The raw data files are `matharena_raw.json` and `putnam_raw.json`.
To process the files, including the automatic labeling of admissible vocabulary given the problem category and answer type (e.g. `Nat`, `Real`, `Set Nat`, not the ground turth answer!), run
```bash
python -m src.preprocess.process_datasets --data_name matharena
python -m src.preprocess.process_datasets --data_name putnam
```

They will produce processed files `matharena.json` and `putnam.json`. Putnam problems that require manual fix are already resolved in Line 134-149 of `src/preprocess/process_datasets.py`

### Admissibility Check
Recall that, when directly using baseline LLM to solve for answer-construction problem with proof, it is vulnerable to spurious ``successful'' proof that is actually trivial with inadmissible answer and circular argument, as in paper Section 5.4. 
The table below compares the standard evaluation metric against a naive metric that only checks Lean compilation. Each entry is `a + b`, where `a` is the number of solved instances under the standard metric, and `b` is the number of spurious solved instances counted only by the naive metric without admissibility checking.

| Method | MathArena | PutnamBench |
| --- | ---: | ---: |
| ECP | 18 + <span style="color:red">0</span> | 17 + <span style="color:red">0</span> |
| GPT-5.4 baseline | 13 + <span style="color:red">19</span> | 8 + <span style="color:red">134</span> |
| Goedel-Prover-V2-32B baseline | 7 + <span style="color:red">18</span> | 7 + <span style="color:red">99</span> |

## Demonstration

To run the admissibility checker demo:

```bash
python -m src.demo.admissibility_dataset_demo
```

The demo selects 10 problems from the datasets, each with admissible and inadmissible answers for comparison.

To run an ECP end-to-end loop on an example with gpt-5.4 for both answer construction and final proof generation, run

```bash
python -m src.demo.ecp_demo --mode ecp \
    --theorem 'theorem P2026AIMEI_1 : ∃ (P2026AIMEI_1_answer : Nat), ∀ (d p : ℚ), (((0 : ℚ) < p) ∧ (d / p = d / (p + (2 :
  ℚ)) + (1 : ℚ)) ∧ (d / (p + (2 : ℚ)) = d / (p + (9 : ℚ)) + (1 : ℚ))) → (∃ (m : ℕ) (n : ℕ), m.Coprime n ∧ d = (↑m : ℚ) /
  (↑n : ℚ) ∧ m + n = P2026AIMEI_1_answer) := by sorry' \
    --output-dir output/demo_gpt54_prover \
    --formalization-demo-dir Formalization/demo \
    --conjecturer-model gpt-5.4 \
    --prover-model gpt-5.4 \
    --baseline mcp \
    --pass-at-n 2 \
    --correction-rounds 2 \
    --show-proof true
```
This takes around 2 minutes for answer construction and 2 minutes for proving.

To run an ECP end-to-end loop on an example with gpt-5.4 for answer construction, and the Goedel-LM/Goedel-Prover-V2-8B for final proof generation, first deploy the model (with at least 1 GPU) by

```bash
bash src/scripts/deploy/deploy_goedel8b.sh
```

python -m src.demo.ecp_demo --mode ecp --theorem 'theorem P2026AIMEI_1 : ∃ (P2026AIMEI_1_answer : Nat), ∀ (d p : ℚ),
  (((0 : ℚ) < p) ∧ (d / p = d / (p + (2 : ℚ)) + (1 : ℚ)) ∧ (d / (p + (2 : ℚ)) = d / (p + (9 : ℚ)) + (1 : ℚ))) → (∃ (m : ℕ)
  (n : ℕ), m.Coprime n ∧ d = (↑m : ℚ) / (↑n : ℚ) ∧ m + n = P2026AIMEI_1_answer) := by sorry' \
    --output-dir output/demo_api_smoke \
    --formalization-demo-dir Formalization/demo \
    --conjecturer-model gpt-5.4 \

The output lean proof is saved to `Formalization/demo/`.
The expected output is

```text
================================================================================
Proof Generation Result
================================================================================
prover_solved: True
proof_source: goedel_pipeline
proof_output_dir: output/P2026AIMEI_1_20260509_122117/proof_generation/prover/P2026AIMEI_1_final/gpt-5
[demo] Wrote assembled proof to Formalization/demo/P2026AIMEI_1_20260509_122352.lean
[demo] Assembled full proof verified by Lean (source=goedel_pipeline).

================================================================================
Assembled Full Proof
================================================================================
import Mathlib
import Aesop
open BigOperators Classical ENNReal Equiv EuclideanGeometry Filter Finset Fintype Function Lex List MeasureTheory Nat ProbabilityTheory Real SimpleGraph Real Nat Topology Rat
set_option maxHeartbeats 0
theorem P2026AIMEI_1 : ∃ (P2026AIMEI_1_answer : Nat), ∀ (d p : ℚ),
  (((0 : ℚ) < p) ∧ (d / p = d / (p + (2 : ℚ)) + (1 : ℚ)) ∧ (d / (p + (2 : ℚ)) = d / (p + (9 : ℚ)) + (1 : ℚ))) → (∃ (m : ℕ)
  (n : ℕ), m.Coprime n ∧ d = (↑m : ℚ) / (↑n : ℚ) ∧ m + n = P2026AIMEI_1_answer) := by
  use (277 : Nat)
  intro d p h
  rcases h with ⟨hp, h1, h2⟩
  have hp0 : p ≠ 0 := by linarith
  have hp2_0 : p + 2 ≠ 0 := by linarith
  have hp9_0 : p + 9 ≠ 0 := by linarith

  have hd1 : d = p * (p + 2) / 2 := by
    apply (eq_div_iff (by norm_num : (2 : ℚ) ≠ 0)).2
    have := h1
    field_simp [hp0, hp2_0] at this
    linarith

  have hd2 : d = (p + 2) * (p + 9) / 7 := by
    apply (eq_div_iff (by norm_num : (7 : ℚ) ≠ 0)).2
    have := h2
    field_simp [hp2_0, hp9_0] at this
    linarith

  have hp_eq : p = (18 : ℚ) / 5 := by
    have hEq : p * (p + 2) / 2 = (p + 2) * (p + 9) / 7 := by
      rw [← hd1, hd2]
    have hEq' := hEq
    field_simp [hp2_0] at hEq'
    linarith

  have hd_eq : d = (252 : ℚ) / 25 := by
    rw [hd1, hp_eq]
    norm_num

  refine ⟨252, 25, ?_, ?_, ?_⟩
  · norm_num
  · simpa using hd_eq
  · norm_num
```

## Experiment Reproduction

We strongly recommend to run the end-to-end experiment reproduction on high-performance computing cluster. The cluster must have internet access on compute node for calling external API and downloading Huggingface models such Goedel-Prover-V2 models. The response generation is implemented by batched vLLM inference, with no need to deploy local servers.

To run the end-to-end experiment reproduction of ECP, first allocate GPU. Then, run the ray initializer

```bash
bash src/sh/run_ray.sh
```

This enables multi-node GPU resources on cluster.

To run the best setup, execute

```bash
python -m src.ecp.main \
    --mode ecp \
    --baseline mcp \
    --problem_path putnam \
    --conjecturer_model GPT-5.4 \
    --prover_model Goedel-LM/Goedel-Prover-V2-32B \
    --pass_at_n 32 \
    --correction_rounds 2
```

For PutnamBench (346 problems), answer-construction with GPT-5.4 will take around 30 USD (~346 responses). The final proof generation with Goedel-LM/Goedel-Prover-V2-32B under Pass@32 and correction_round = 2 creates maximum of 346 * 32 * 3=33216 responses in total. The estimated compute is around 250 hours for H100 (80GB) or equivalently 1200 hours for L40S (48), including around 25 hours for Lean proof compilation with CPUs.

Other setups (where `problem_path in ['putnam', 'matharena']`, `prover_model in ['Goedel-LM/Goedel-Prover-V2-8B', 'Goedel-LM/Goedel-Prover-V2-32B']`, `baseline in ['mcp', 'cot']`) can be executed by changing the `problem_path/prover_model/baseline` in the above command.

For baseline commercial LLM's end-to-end proof generation, run

```bash
python -m src.ecp.main \
    --mode llm_baseline \
    --problem_path putnam \
    --prover_model GPT-5.4 \
    --pass_at_n 4 \
    --correction_rounds 2

python -m src.ecp.main \
    --mode llm_baseline \
    --problem_path matharena \
    --prover_model GPT-5.4 \
    --pass_at_n 32 \
    --correction_rounds 2
```
This will cost around 180 (putnam, pass@4) + 340 (matharena, pass@32) = 520 USD OpenAI API. Using putnam with pass@32 will cost around 180*(32/4)=1440 USD API.
